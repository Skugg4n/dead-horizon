# Dead Horizon -- Polish Sprint Plan

## User feedback themes (repeated issues):
1. Panel overflow -- equipment panel STILL doesn't paginate properly in all cases
2. Consistency -- glass shards glow vs other traps don't, font sizes vary
3. Discoverability -- refugees, trap combos, systems not explained
4. Navigation -- too many clicks to scroll through 20+ items
5. Visual polish -- post-apocalyptic feel needs work, lighting, color
6. Balance -- some traps much better than others, early game limited
7. Depth -- trap combos, synergies not leveraged

## Sprint phases:

### Phase 1: EVALUATE (research + analysis)
- [ ] Audit all panels for overflow/clipping issues
- [ ] Audit trap balance (damage per scrap cost ratio)
- [ ] Audit build menu categories (count items per tab)
- [ ] Audit visual consistency (glow effects, colors, fonts)
- [ ] Research post-apocalyptic color palettes
- [ ] Research trap combo mechanics in similar games

### Phase 2: PLAN (design solutions)
- [ ] Design page-based navigation for long lists
- [ ] Design trap combo system
- [ ] Design improved color palette
- [ ] Design pickup notification visuals
- [ ] Design refugee tutorial/explanation
- [ ] Plan trap rebalancing

### Phase 3: IMPLEMENT (code changes)
- [ ] Fix ALL panel overflow with page-based nav
- [ ] Implement trap combos (knockback -> damage zone)
- [ ] Apply post-apocalyptic color grading
- [ ] Add pickup ping/notification visuals
- [ ] Rebalance trap categories
- [ ] Add refugee explanation in UI
- [ ] Standardize glow/visual effects on all traps
- [ ] Fix starting trap availability

### Phase 4: VERIFY (test + evaluate)
- [ ] Screenshot every panel with max content
- [ ] Test trap combo in gameplay
- [ ] Verify color palette looks good
- [ ] Run full playthrough Forest -> City
- [ ] Check all text fits in panels
